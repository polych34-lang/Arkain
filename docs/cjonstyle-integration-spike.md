# CJ온스타일 표준 API 통합 스파이크 노트 (ARK-45)

**상태:** 코드 구현 시작 불가 — **입점/계약 전제조건 미충족(블로커)** · 공개 정보만 확인 완료
**대상:** CJ온스타일 (표준 API, `partners.cjonstyle.com`)
**소유자:** Founding Engineer · **이슈:** [ARK-45](/ARK/issues/ARK-45) · **날짜:** 2026-07-03

이 문서는 ARK-3(네이버)·ARK-27(쿠팡)과 같은 목적의 통합 노트지만, 이번엔 "구현
완료 + 라이브 검증 대기"가 아니라 **"구현 착수 자체가 막힘"** 상태를 기록한다. 이유는
§2 참고. `docs/tv-homeshopping-integration-spike.md`(ARK-15) §5의 후속이다.

---

## 1. 왜 다른가 (네이버/쿠팡/ESM과 비교)

| | 네이버(ARK-3) / 쿠팡(ARK-27) / ESM(ARK-11) | CJ온스타일(ARK-45) |
| --- | --- | --- |
| API 스펙 문서 | 로그인 없이 공개 (또는 사전 확보) | **파트너 로그인 후에만 열람** — 필드/페이징/에러코드 전부 비공개 |
| 어댑터 코드 | 스펙 기반으로 먼저 구현 + 모킹 테스트 통과, 라이브만 자격증명 대기 | **작성 불가** — 실제 요청/응답 필드를 모르는 상태에서 코드를 쓰면 추측(fabrication)이 되고, 이는 "셀러 데이터 정확성" 우선순위(AGENTS.md #2)에 정면으로 위배 |
| 블로커 성격 | 라이브 검증만 자격증명 대기 | **입점 자체가 API 신청의 전제조건** — 계약 전에는 테스트 계정도 없음 |

즉 CJ온스타일은 "코드 다 짜놓고 자격증명만 기다리는" 패턴이 아니라, **입점 확인 →
API 스펙 확보 → 그제서야 어댑터 설계 착수**의 순서를 강제한다.

---

## 2. 오늘 재확인한 공개 정보 (로그인 없이 확인 가능한 범위)

- `partners.cjonstyle.com/standardApi/apiGuide`: 절차가 "협력사 입점 → 파트너시스템
  로그인 → API 정보관리"로 명시됨. **입점하지 않은 업체의 테스트 계정 신청 경로는
  없음** (셀프서비스 불가 — 네이버 SELF 앱처럼 스스로 발급받을 방법이 없다).
- IP 화이트리스트: 파트너시스템 > API 관리 > API 정보관리 메뉴에서 등록. 협력사의
  개발서버/운영서버 IP를 각각 등록해야 함.
- `partners.cjonstyle.com/standardApi/apiDocs`: 문서 틀만 공개. 인증 헤더 필드명
  일부 노출(`vendorCode`, `authenticationKey`) — 그러나 실제 주문(order) 엔드포인트의
  요청/응답 스키마, 페이징 방식, 에러 코드는 **비공개**.
- API 카테고리 5개(ARK-15 §1 재확인): 공통 / 협력사 / 상품 / **주문·배송** / 고객관리.

이 정보만으로는 `NormalizedOrder` 매핑 설계조차 확정할 수 없다 — 필드명을 모른다.

## 3. 입점 후 착수할 작업 (선반영 계획, 코드 없음)

CEO/보드가 입점을 확정하고 API KEY + IP 화이트리스트를 발급받으면, 다음 순서로
ARK-3/ARK-27과 동일한 어댑터 패턴을 적용한다:

1. 파트너시스템 로그인 후 주문·배송 API 실 스펙(필드/페이징/레이트리밋/에러코드) 확보.
2. `docs/tv-homeshopping-integration-spike.md` §2의 TV홈쇼핑 도메인 차이(방송 편성
   버스트, ARS 주문 비중, 방송회차 단위 상품)를 실제 API 응답과 대조해 매퍼 설계에
   반영 — `orderedAt`을 "방송 시각"과 "주문 확정 시각" 중 무엇으로 정규화할지는 실
   응답을 봐야 결정 가능.
3. `src/integrations/cjonstyle/` 하위에 `cjonstyle.http.ts`(인증: API KEY 헤더 +
   IP 화이트리스트 — bcrypt 서명이나 HMAC 불필요할 가능성, 스펙 확인 후 확정) /
   `cjonstyle.types.ts` / `cjonstyle.mapper.ts` / `cjonstyle.adapter.ts` 구현,
   `test/cjonstyle.test.ts` 모킹 테스트.
4. `MarketplaceId`에 `"cjonstyle"` 추가, `main.ts` 어댑터 맵 등록, `npm run
   cjonstyle:pull` 스크립트 — 쿠팡(ARK-27) 때와 동일하게 코어 미변경.

공수 추정은 ARK-15 §4 유지: 3~5일 (스펙 확인 후 재산정 필요).

## 4. 블로커 (CEO/보드 액션 필요)

에이전트가 대신 할 수 없는 실제 사람/회사 액션이다 — CJ온스타일 입점은 사업자
등록·매출 실적 등 회사 단위 심사가 필요한 절차로 추정되며, 에이전트가 대리 신청할
권한이 없다(AGENTS.md 경계: "실 셀러 자격증명/운영 마켓 계정은 CEO 승인 없이 건드리지
않는다"). 상세 액션 항목은 별도 이슈 [ARK-47](/ARK/issues/ARK-47)에 분리했다.

핵심 확인 필요 사항:
1. ARKAIN(또는 테스트 목적 협력사)이 이미 CJ온스타일 입점 상태인지.
2. 아니라면 입점을 지금 추진할지 여부(ARK-15 §5에서 CJ온스타일을 1순위로 추천했고
   이미 보드가 이 스파이크를 승인해 착수시켰으므로, 우선순위 자체는 재확인 불필요 —
   남은 건 "입점 절차 실행"뿐).
3. 입점 후 파트너시스템 로그인 → API 정보관리에서 API KEY 발급 + IP 화이트리스트
   등록 + 주문·배송 API 스펙 문서 확보(스크린샷 또는 PDF 공유만으로도 어댑터 설계
   착수 가능 — 실 크리덴셜 전달보다 먼저 스펙만 공유해도 됨).
