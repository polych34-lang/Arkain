# SellerDesk × 네이버 스마트스토어 연동 가이드 (셀러 셀프서비스)

**대상 독자:** SellerDesk를 쓰는 셀러(비개발자) · **소요:** 약 5~10분
**용도:** 이 문서는 일회성 지원용이 아니라 **제품 온보딩 화면/도움말에 재사용**하도록
작성됐습니다. 각 단계는 온보딩 UI의 한 스텝에 대응합니다(§7 제품팀 노트 참고).
**연동 방식(현재):** 네이버 커머스 API — 앱 타입 **SELF**.

> **문서 재사용 규칙:** `[화면]` 표시는 온보딩 UI 스크린샷/일러스트가 들어갈 자리입니다.
> 셀러용 문구는 그대로 카피로 쓸 수 있게 존댓말·평이한 용어로 작성했습니다. 개발/보안
> 세부는 `> 개발 노트:` 블록으로 분리했으니 셀러 화면에서는 접거나 생략하세요.

---

## 1. 시작 전 준비물

- 네이버 **커머스 계정** (스마트스토어를 운영 중인 계정)
- 운영 중인 **스마트스토어** 1개 (또는 테스트용 신규 스토어)
- 휴대폰 본인인증 (네이버 정책상 필요할 수 있음)

준비되면 아래 5단계만 따라오시면 됩니다.

---

## 2. 왜 연동이 필요하고, 내 정보는 안전한가요?

SellerDesk가 여러 마켓의 주문·상품·정산을 한 화면에서 관리하려면, 네이버에서 발급하는
**API 열쇠(client_id / client_secret)** 가 필요합니다. 이 열쇠로 SellerDesk가 셀러님을
대신해 주문/상품 데이터를 안전하게 읽어옵니다.

**SellerDesk의 자격증명 보관 방식(안심하세요):**
- 열쇠는 저장 전에 **AES-256-GCM으로 암호화**되어, 설령 데이터베이스가 유출돼도 열쇠는
  풀리지 않습니다. (암호화 키는 DB가 아닌 별도 시크릿 매니저에 보관)
- 동기화가 실제로 필요한 **그 순간에만** 잠깐 복호화해서 쓰고, 평소엔 평문으로 두지 않습니다.
- 최소 권한: 주문/상품 조회에 필요한 범위만 사용합니다.

> 개발 노트: `src/secrets/credentialStore.ts`(봉투 암호화) + `ARCHITECTURE.md §7`.
> 열쇠는 로그에도 남지 않도록 redaction 처리됨(`src/logging/logger.ts`).

## 3. 단계별 연동 (SELF 방식)

### Step 1 — 커머스 API 센터 접속 `[화면]`
1. [커머스 API 센터](https://apicenter.commerce.naver.com)에 접속합니다.
2. 스마트스토어를 운영하는 **네이버 계정으로 로그인**합니다. (본인인증 요구될 수 있음)

### Step 2 — 애플리케이션 등록 (타입: SELF) `[화면]`
1. **애플리케이션 등록/신청** 메뉴로 이동합니다.
2. 유형은 **SELF**(내 스토어용 앱)를 선택합니다.
3. 연동할 **내 스마트스토어를 선택/연결**합니다.
4. (테스트용이라면) 실제 판매 데이터 없이 더미 상품 1~2개, 더미 주문 1~2건만 있어도
   연동 확인에 충분합니다.

### Step 3 — 열쇠 확인 `[화면]`
등록이 끝나면 **`client_id`(애플리케이션 ID)** 와 **`client_secret`(시크릿)** 이
발급됩니다. 이 두 값을 복사해 둡니다.

> ⚠️ **시크릿은 채팅·이슈·메일 등 공개된 곳에 붙여넣지 마세요.** 아래 SellerDesk
> 연동 화면에만 입력합니다.

### Step 4 — SellerDesk에 입력 `[화면: 연동하기]`
1. SellerDesk → **연동 관리 → 네이버 스마트스토어 → 연동하기**로 이동합니다.
2. `client_id`와 `client_secret`을 각 칸에 붙여넣고 **저장**합니다.
3. 저장 즉시 SellerDesk가 열쇠를 **암호화해 보관**합니다(§2).

### Step 5 — 연동 확인 & 첫 동기화 `[화면: 연동 완료]`
- SellerDesk가 자동으로 **열쇠 유효성**을 확인하고(성공 시 초록 배지), 최근 주문/상품을
  가져오기 시작합니다.
- 완료되면 대시보드에 네이버 주문/상품이 나타납니다.

> 개발 노트: 유효성 확인 = `NaverSmartstoreAdapter.verifyCredential`(토큰 발급 성공
> 여부). 첫 동기화 = `npm run naver:pull`이 수행하는 것과 동일한 경로
> (`fetchOrders` + `fetchProducts`), 결과는 로컬/DB에 멱등 upsert.

## 4. 잘 안 될 때 (FAQ)

| 증상 | 원인 / 해결 |
| --- | --- |
| "연동 실패 / 인증 오류" | `client_id`/`secret` 오타 또는 공백 혼입. 다시 복사해 붙여넣기. |
| "권한 없음" | Step 2에서 스토어 연결이 안 됨. API 센터에서 앱↔스토어 연결 재확인. |
| "잠시 후 다시" 안내 | 네이버 호출 한도(레이트리밋). SellerDesk가 자동 재시도하니 기다리면 됩니다. |
| 주문이 안 보임 | 조회 기간(기본 최근 14일) 밖이거나 해당 기간 주문이 없음. |

> 개발 노트: 오류 매핑/재시도는 `mapHttpError` + `retry.ts`에 일원화. 429/5xx/RATELIMIT만
> 재시도, `Retry-After` 존중. 자세한 API 쿼크는 `docs/naver-commerce-integration.md`.

## 5. 보안 요약 (한 번 더)

- 열쇠는 **암호화 저장**, 사용 순간에만 복호화, 로그 미노출.
- SellerDesk는 셀러의 **주문/상품 조회**에 필요한 범위만 사용합니다.
- 열쇠 교체가 필요하면 API 센터에서 재발급 후 Step 4만 다시 하면 됩니다.

---

## 6. 나중에: 매장이 여러 개거나 마켓이 늘면

지금(SELF) 방식은 스토어별로 위 절차를 반복합니다. SellerDesk가 지원 마켓(쿠팡·11번가
등)을 늘려도 셀러 입장의 절차는 동일한 모양(열쇠 발급 → 연동 화면 입력)입니다. 마켓별
세부 화면만 달라집니다.

## 7. 제품팀 노트 (재사용 & 확장 설계)

**이 가이드 ↔ 온보딩 UI 매핑**
- §3의 Step 1~5를 그대로 온보딩 마법사 5스텝으로 사용. Step 4가 유일한 SellerDesk 입력
  화면, 나머지는 안내 + 딥링크(외부 네이버 페이지 새 탭).
- §2/§5 보안 문구는 "연동하기" 화면의 신뢰 카피로 재사용.
- §4 FAQ는 실패 상태의 인라인 도움말로 재사용.

**확장 권고 — 진짜 셀프서비스는 SELLER(솔루션 제공자) 모델**
SELF 방식은 셀러가 **직접 앱을 등록**해야 해서 온보딩 마찰이 큽니다(Step 1~3). 제품
스케일의 매끄러운 셀프서비스는 **SellerDesk를 네이버 커머스 솔루션 제공자로 1회 등록**하고,
셀러는 앱 등록 없이 **"SellerDesk 연결 허용"만 클릭**하면 되는 SELLER 모델입니다. 그러면
셀러 절차는 사실상 "네이버 로그인 → 허용 → 끝"으로 줄어듭니다.
- 어댑터는 **이미 SELLER 모드를 지원**합니다(`accountId` 있으면 `type=SELLER`로 토큰 발급).
  즉 코어 재작성 없이 연결 플로우 + `account_id` 수집만 추가하면 됩니다.
- 전제: SellerDesk의 사업자 명의로 네이버 솔루션 등록/심사(사람·법무 단계)가 필요합니다.
- 이 전환은 스파이크(ARK-3) 범위를 넘는 **제품 기능**이라 별도 이슈로 추적합니다.

**업데이트 (ARK-21, 구현 완료):** 위 SELLER 모델 전환이 코드로 구현됐습니다.
`/onboarding/naver`(5스텝 마법사)와 `/connections`(연동 상태/재연동 관리) 화면,
`connectNaverSeller` 연결 서비스, `POST /api/connections/naver/connect` +
`GET /api/connections`. 자세한 내용과 잔여 블로커는
`docs/naver-seller-selfservice.md` 참고.
